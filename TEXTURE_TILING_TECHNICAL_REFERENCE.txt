TECHNICAL EXPLANATION:
======================

Texture Tiling Parameter (texTiling):
-------------------------------------
The texTiling parameter controls how many times a texture repeats across a 3D 
surface. It's passed to the shader which multiplies the texture coordinates.

In the shader (fragment shader):
    vec2 adjustedTexCoord = texCoord * texTiling;
    vec4 texColor = texture(texture1, adjustedTexCoord);

Example:
- texTiling = 1.0: texture displays exactly once across the surface
- texTiling = 2.0: texture repeats twice (2x2 grid = 4 tiles visible)
- texTiling = 4.0: texture repeats 4 times (4x4 grid = 16 tiles visible)
- texTiling = 25.0: texture repeats 25 times (25x25 grid = 625 tiles!)

The Problem with High Tiling:
-----------------------------
When a 512x512 pixel texture is tiled 25 times across a wall:
- Each "brick" in the texture becomes (512/25) = ~20 pixels
- At normal viewing distance, these pixels become clearly visible
- Individual pixels look like blocks ? pixelated appearance
- User sees a blurry, distorted pattern instead of realistic bricks

Mathematical Analysis:
=====================

Surface Area vs Texture Resolution:
----------------------------------

North Wall Example:
  Surface dimensions: 50 units wide ū 5 units tall = 250 square units
  Texture image: 512ū512 pixels = 262,144 pixels total
  
  With tiling = 25.0:
    Effective texture scale: 262,144 pixels / (50 ū 25) = ~209 pixels wide
    ? Each "column" of bricks gets ~209 pixels
    ? Visible brick size: ~8 pixels (too small, appears as noise)
    
  With tiling = 2.0:
    Effective texture scale: 262,144 pixels / (50 ū 2) = ~2621 pixels wide
    ? Each "column" of bricks gets ~2621 pixels
    ? Visible brick size: ~100 pixels (realistic detail level)

The Golden Rule of Texturing:
-----------------------------
texTiling should be chosen so that 1 "feature" (one brick) appears to be:
- Minimum 40-50 pixels on screen (detail threshold)
- Maximum 256-300 pixels on screen (proper scale)

At typical viewing distance (5-10m), our chosen values achieve:
- texTiling = 1.0-2.0 ? 100-200 pixels per brick ? PERFECT
- texTiling = 4.0-12.0 ? 50-100 pixels per feature ? GOOD (outdoor)
- texTiling = 25.0+ ? 20-50 pixels per brick ? TOO SMALL

SHADER INTEGRATION:
===================

Fragment Shader Processing:
--------------------------
The texture coordinates passed from the vertex shader are multiplied by 
texTiling in the fragment shader:

    // In fragment shader
    in vec2 TexCoord;
    uniform float texTiling;
    uniform sampler2D texture1;
    
    void main() {
        vec2 adjustedCoord = TexCoord * texTiling;
        vec4 texColor = texture(texture1, adjustedCoord);
        // Use texColor for final pixel color
    }

With GL_REPEAT wrap mode:
- Coordinates wrap around after 1.0, creating seamless repetition
- No visible seams between adjacent tiles (if brick.jpg is tileable)

Why Linear Filtering Helps:
----------------------------
At low tiling values, the texture isn't being minified (reduced), so:
- GL_LINEAR_MIPMAP_LINEAR provides smooth interpolation
- Mipmap chains ensure good quality at any distance
- No aliasing artifacts

PERFORMANCE IMPACT:
===================

This change has ZERO performance impact because:
? Texture tiling is computed in the vertex/fragment shaders
? GPU processes tiling mathematically (multiplication)
? No additional geometry or draw calls
? Same GPU bandwidth requirements
? Identical fill rate usage

Memory usage unchanged:
? Same texture file loaded (brick.jpg)
? Same GPU texture memory allocated
? Only the UV coordinate multiplication changes

REFERENCE VALUES FOR FUTURE TEXTURING:
======================================

Recommended tiling values by surface type:

Architectural/Masonry:
  - Large facade walls:     1.0 - 2.0
  - Medium walls (< 10m):   2.0 - 3.0
  - Small accent walls:     3.0 - 5.0

Industrial/Metal:
  - Large panels:           2.0 - 4.0
  - Machine surfaces:       4.0 - 8.0
  - Small components:       6.0 - 12.0

Organic/Natural:
  - Ground/terrain:         4.0 - 12.0
  - Stone surfaces:         2.0 - 6.0
  - Wood surfaces:          2.0 - 4.0

General Formula:
  Optimal_Tiling = DesiredPixelSize / TextureResolution
  
  Example: Want 100 pixels per brick, using 512px texture
  Optimal_Tiling = 512 / 100 = ~5.0
